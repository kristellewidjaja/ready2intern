# PRD Generator Prompt

Generate a comprehensive Product Requirements Document (PRD) for the following project.

## PROJECT DESCRIPTION:
[Describe your project here - what you want to build, the problem it solves, who it's for]

## ADDITIONAL CONTEXT (Optional):
[Add any additional details: target users, constraints, timeline, budget, tech stack preferences, integrations needed, etc.]

---

Please create a well-structured PRD that includes:

1. **Executive Summary**: Brief overview of the project (2-3 paragraphs)

2. **Problem Statement**: 
   - What problem are we solving?
   - Why is it important?
   - Current alternatives and their limitations

3. **Goals & Objectives**: 
   - Clear, measurable goals
   - Success criteria
   - What we're NOT trying to achieve (out of scope)

4. **Target Users**: 
   - Primary user personas
   - User characteristics and needs
   - User segments if applicable

5. **User Stories**: 
   - Key user scenarios (As a [user], I want to [action], so that [benefit])
   - User journey/workflow
   - Edge cases to consider

6. **Functional Requirements**: 
   - Core features (must-have)
   - Secondary features (should-have)
   - Future considerations (nice-to-have)
   - Feature prioritization

7. **Non-Functional Requirements**:
   - Performance expectations
   - Security requirements
   - Scalability needs
   - Accessibility standards
   - Compliance requirements

8. **Technical Considerations**:
   - Suggested architecture approach
   - Technology stack recommendations
   - Integration points
   - Data model considerations
   - Infrastructure needs

9. **User Interface/Experience**:
   - Key UI/UX principles
   - Design requirements
   - Platform considerations (web, mobile, desktop)
   - Accessibility requirements

10. **Success Metrics**:
    - KPIs to track
    - How we'll measure success
    - Analytics requirements
    - Monitoring and reporting needs

11. **Timeline & Milestones**:
    - Suggested project phases
    - Key milestones
    - Dependencies
    - Launch criteria

12. **Risks & Mitigation**:
    - Technical risks
    - Business risks
    - Resource risks
    - Mitigation strategies

13. **Open Questions**:
    - Items requiring further discussion
    - Decisions to be made
    - Areas needing research

Format the output in clean markdown with proper headings, bullet points, and tables where appropriate. Be specific and actionable. Keep it concise but comprehensive.
